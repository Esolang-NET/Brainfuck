# BF0007

Brainfuck source contains output command `.` but the target method has no supported output destination.

## Trigger

This diagnostic is reported when the source requires output and the method signature does not provide any of the following:

- `string`
- `Task<string>`
- `ValueTask<string>`
- `IEnumerable<byte>`
- `IAsyncEnumerable<byte>`
- `System.IO.Pipelines.PipeWriter`

`void`, `Task`, and `ValueTask` are only valid for output-producing sources when a `PipeWriter` parameter is present.

## Example

```cs
using Esolang.Brainfuck;

partial class Sample
{
	[GenerateBrainfuckMethod("+.")]
	public static partial void Invalid();
}
```

## Fix

Change the return type to a supported output type or add a `System.IO.Pipelines.PipeWriter` parameter.
