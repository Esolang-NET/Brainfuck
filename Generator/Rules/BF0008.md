# BF0008

Brainfuck source contains input command `,` but the target method has no supported input source.

## Trigger

This diagnostic is reported when the source requires input and the method signature does not provide either of the following parameters:

- `string`
- `System.IO.Pipelines.PipeReader`

The return type does not affect this rule. Any supported return type still requires one supported input parameter when the source contains `,`.

## Example

```cs
using Esolang.Brainfuck;

partial class Sample
{
	[GenerateBrainfuckMethod(",")]
	public static partial Task<string> Invalid();
}
```

## Fix

Add either a `string` parameter or a `System.IO.Pipelines.PipeReader` parameter.
